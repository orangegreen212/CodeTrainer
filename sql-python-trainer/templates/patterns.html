🧮 Aggregation Patterns

Pattern: Count/Sum/Avg by Group

1

Identify the metric you need (COUNT, SUM, AVG, MAX, MIN)

2

Find the grouping column (by user, by date, by category)

3

Write SELECT with GROUP BY

Example: Average endorsements per user

SELECT AVG(user_count) AS avg_endorsements
FROM (
    -- Step 1: Count endorsements per user
    SELECT user_id, COUNT(endorsement_id) AS user_count
    FROM fct_skill_endorsements e
    JOIN dim_skills s ON e.skill_id = s.skill_id
    WHERE s.skill_category = 'TECHNICAL'
      AND e.endorsement_date BETWEEN '2024-08-01' AND '2024-08-31'
    GROUP BY user_id
) AS user_stats;

How it works:

  1. First, join tables and filter by category and date
  2. Then group by user_id to count endorsements per person
  3. Finally, calculate AVG of those counts in outer query

🪟 Window Function Patterns

Pattern: Ranking within Groups

1

Choose rank function: ROW_NUMBER (unique), RANK (with gaps), DENSE_RANK (no gaps)

2

PARTITION BY the grouping column

3

ORDER BY the ranking criteria

4

Filter in outer query (WHERE rnk = 1)

Example: Top 2 orders per country

WITH RankedOrders AS (
    SELECT 
        u.country,
        o.order_id,
        o.amount,
        DENSE_RANK() OVER (
            PARTITION BY u.country 
            ORDER BY o.amount DESC
        ) as rnk
    FROM orders o
    JOIN users u ON o.user_id = u.user_id
)
SELECT country, order_id, amount
FROM RankedOrders
WHERE rnk <= 2;

🔗 JOIN Patterns

Pattern: Which JOIN to use?

  • INNER JOIN: Only matching records from both tables
  • LEFT JOIN: All from left table + matching from right (use for "users without X")
  • RIGHT JOIN: All from right table + matching from left (rarely used)
  • FULL OUTER JOIN: All records from both tables

Example: Users without orders

SELECT u.user_id
FROM users u
LEFT JOIN orders o ON u.user_id = o.user_id
WHERE o.order_id IS NULL;

LEFT JOIN keeps all users, WHERE filters to those with no orders

📝 CTE (WITH) Patterns

When to use CTEs

  • Breaking complex queries into readable steps
  • Calculating intermediate results
  • Avoiding subquery repetition
  • Multi-step calculations (DAU → MAU → Sticky Factor)

Example: DAU & Sticky Factor

WITH dau AS (
    SELECT 
        DATE(event_time) AS dt,
        COUNT(DISTINCT user_id) AS daily_users
    FROM events
    WHERE event_time >= '2024-01-01'
    GROUP BY 1
),
mau AS (
    SELECT COUNT(DISTINCT user_id) AS monthly_users
    FROM events
    WHERE event_time >= '2024-01-01'
)
SELECT 
    dau.dt,
    dau.daily_users,
    mau.monthly_users,
    (dau.daily_users * 100.0 / mau.monthly_users) AS sticky_factor
FROM dau
CROSS JOIN mau;

🐼 Pandas Patterns

Pattern: GroupBy → Aggregate → Analyze

1

Load data: pd.read_csv() or df provided

2

Group by column: df.groupby('column')

3

Aggregate: .sum(), .mean(), .count(), .nunique()

4

Reset index: .reset_index() for clean output

Example: Revenue by platform

# Step 1: Join tables
df_merged = df_users.merge(df_orders, on='user_id')

# Step 2: GroupBy and Sum
result = df_merged.groupby('platform')['amount'].sum().reset_index()

# Step 3: Display
print(result)
