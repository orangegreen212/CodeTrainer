Ranking & Window Functions

Six patterns for tasks #1–57 using SQL window functions. Used for ranking, time gaps, running totals, rolling averages, first/last values, and percentiles.

Pattern 1: Ranking within Groups (Top N)

Tasks #4, #11, #31, #34, #37, #41, #94, #95
1

Choose rank function: ROW_NUMBER (unique), RANK (gaps on ties), DENSE_RANK (no gaps)

2

PARTITION BY: grouping column (e.g. country, user_id)

3

ORDER BY: sorting criterion (e.g. amount DESC)

4

Filter: outer query WHERE rnk <= N

-- Task #95: Top 2 orders per country
WITH RankedOrders AS (
    SELECT u.country, o.order_id, o.amount,
           DENSE_RANK() OVER (
               PARTITION BY u.country
               ORDER BY o.amount DESC
           ) as rnk
    FROM orders o JOIN users u USING(user_id)
)
SELECT country, order_id, amount
FROM RankedOrders WHERE rnk <= 2;

Pattern 2: Time Gap Analysis (Difference between Events)

Tasks #37, #39, #42, #48, #51
1

Use LAG/LEAD: get value from previous or next row

2

PARTITION BY user_id: analyze one user's history

3

ORDER BY time: always by event_time or order_date

4

Calculate Gap: subtract previous time from current in outer query

-- Task #37: Gap between 1st and 2nd purchase
SELECT user_id,
       order_date - prev_order_date as gap_days
FROM (
    SELECT user_id, order_date,
           LAG(order_date) OVER (
               PARTITION BY user_id
               ORDER BY order_date
           ) as prev_order_date,
           ROW_NUMBER() OVER (
               PARTITION BY user_id
               ORDER BY order_date
           ) as order_num
    FROM orders
) t
WHERE order_num = 2;

Pattern 3: Running Totals (Cumulative Sum)

Tasks #13, #43, #47, #57, #73
1

Use SUM/COUNT or any aggregate

2

PARTITION BY if you need per-user running total

3

ORDER BY determines accumulation order (usually by time)

-- Task #47: Cumulative user spend
SELECT user_id, order_date, amount,
       SUM(amount) OVER (
           PARTITION BY user_id
           ORDER BY order_date
       ) as running_total_spend
FROM orders;

Pattern 4: Rolling Windows (Moving Averages)

Tasks #2, #10, #38, #45
1

Use AVG/SUM for trend smoothing

2

ORDER BY date

3

Define Frame: ROWS BETWEEN N PRECEDING AND CURRENT ROW

-- Task #38: 7-day rolling average revenue
SELECT order_date,
       AVG(SUM(amount)) OVER (
           ORDER BY order_date
           ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
       ) as rolling_avg_7d
FROM orders
GROUP BY 1;

Pattern 5: First/Last Value Attribution

Tasks #14, #35, #52, #56
1

Use FIRST_VALUE / LAST_VALUE to carry a value across all rows of a user

2

PARTITION BY user_id

3

Frame Fix for LAST_VALUE: always add RANGE BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING

-- Task #56: Last UTM source before purchase
SELECT DISTINCT user_id,
       LAST_VALUE(utm_source) OVER (
           PARTITION BY user_id
           ORDER BY event_time
           ROWS BETWEEN UNBOUNDED PRECEDING
               AND UNBOUNDED FOLLOWING
       ) as last_attribution_source
FROM events;

Pattern 6: Distribution and Percentiles

Tasks #19, #49, #57, #69
1

Use PERCENT_RANK / NTILE to divide sample into parts

2

ORDER BY the metric (e.g. total_spend)

3

Filter to find anomalies or VIP customers

-- Task #49: Top 1% Spenders (Pareto)
WITH UserSpending AS (
    SELECT user_id, SUM(amount) as total_spent,
           PERCENT_RANK() OVER (
               ORDER BY SUM(amount) DESC
           ) as pct_rank
    FROM orders
    GROUP BY 1
)
SELECT * FROM UserSpending
WHERE pct_rank <= 0.01; -- Top 1%

Exclusion & Conversion Patterns

Find missing records and calculate conversion rates — two of the most common real interview questions.

Pattern 7: The "Exclusion" Pattern (Finding the Missing)

Tasks #12, #16, #32
1

LEFT JOIN: take main table (users) and join the actions table (orders)

2

JOIN Key: by user_id

3

Filter NULL: WHERE orders.order_id IS NULL to find non-buyers

4

Result: users for reactivation campaigns

-- Task #12: Users without orders
SELECT u.user_id
FROM users u
LEFT JOIN orders o ON u.user_id = o.user_id
WHERE o.order_id IS NULL;

Pattern 8: Conversion Funnel (The Ratio Pattern)

Tasks #16, #32, #40, #50, #63, #93
1

Count Step A: number of users at the top of funnel

2

Count Step B: number of users who completed the action

3

Multiply by 100.0: avoid integer division in SQL

4

Divide: Step B / Step A

-- Task #16: USA Conversion %
SELECT
    COUNT(DISTINCT o.user_id) * 100.0
    / COUNT(DISTINCT u.user_id) as conv_rate
FROM users u
LEFT JOIN orders o USING(user_id)
WHERE u.country = 'USA';

Cohorts & Data Cleaning

Retention analysis via self-joins and normalizing raw data using SQL or Python.

Pattern 9: The "Self-Join" for Cohorts (Time Offset)

Tasks #36, #44, #67, #97
1

Table A: registrations or first visit

2

Table B: repeated events

3

JOIN Condition: A.user_id = B.user_id

4

Time Condition: B.date = A.date + interval 'N days'

-- Task #36: 7-day Retention
SELECT DATE(u.reg_date) as cohort,
       COUNT(DISTINCT e.user_id) as returned_users
FROM users u
JOIN events e ON u.user_id = e.user_id
WHERE DATE(e.event_time) = DATE(u.reg_date) + 7
GROUP BY 1;

A/B Testing & ML Preprocessing

Statistical significance testing for A/B experiments and preparing data for machine learning models.

Pattern 11: Statistical Significance (The A/B Test Pattern)

Tasks #66, #96
1

Split Groups: filter data into Group A and Group B

2

Extract Metric: select the target column (e.g. revenue)

3

Choose Test: T-test (for means) or Chi-square (for conversions)

4

Interpret P-value: if p < 0.05, result is significant

# Task #66: T-test in Python
from scipy import stats

# 1. Split groups
rev_a = df[df['group'] == 'A']['revenue']
rev_b = df[df['group'] == 'B']['revenue']

# 2. Run t-test
t_stat, p_val = stats.ttest_ind(rev_a, rev_b)

# 3. Decision
is_significant = p_val < 0.05
print(f'p={p_val:.4f}, significant={is_significant}')
